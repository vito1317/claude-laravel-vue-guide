# 鴻準專案：設備維保流程規範

本文件定義了模具維保專案中的兩大核心工作流與狀態轉換邏輯。

## 1. 設備保養流程 (Maintenance Workflow)
適用於週期性保養任務。

### 流程階段與狀態 (States)
1. **Planned (計畫中)**: 透過「排配保養計畫」自動或手動生成。
2. **Assigned (已排配)**: 經由「人員排配」指派給特定人員。
3. **In Progress (執行中)**: 操作員於平板端開始填寫保養表。
4. **Pending Approval (待簽核)**: 填寫完成，發起電子簽核。
5. **Completed (已結案)**: 部門主管審核通過。

### 角色與執行頻率
- **一般操作員 (Operator)**: 負責執行 **日、週** 保養。
- **維修人員 (Technician)**: 負責執行 **月、半年、年** 保養。
- **部門主管 (Manager)**: 負責 **審核 (Approve)** 所有的保養單據。

---

## 2. 設備維修流程 (Repair Workflow)
適用於突發性設備故障或修模需求。

### 流程階段與狀態 (States)
1. **Requested (申請中)**: 由一般操作員發起「設備叫修申請」。
2. **Under Review (審核中)**: 現場主管或部門主管進行初步審核。
3. **Repairing (維修中)**: 維修人員接單並執行維修。
4. **Pending Satisfaction (待滿意度回饋)**: 維修完成，需由用戶填寫滿意度。
5. **Pending Archiving (待歸檔)**: 經過最後的歸檔審核。
6. **Closed (已結案)**: 流程完成。

### 角色與權限
- **一般操作員**: 發起申請、填寫滿意度。
- **現場主管**: 進行初步審核。
- **維修人員**: 執行維修內容填寫。
- **部門主管**: 進行最終審核與歸檔審核。
