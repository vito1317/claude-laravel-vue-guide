# 鴻準專案：iPad 端前端開發標準 (UI/UX Standards)

本專案主要運行於 11 吋 iPad 平板，所有前端介面必須符合觸控優化與現場作業環境的需求。

## 1. 觸控優化規範 (Touch Optimization)

### 按鈕與交互元件
- **最小點擊區域**：所有按鈕與可點擊項目的高度/寬度不得小於 **48px $\times$ 48px**，以確保現場作業人員能精準點擊。
- **間距 (Spacing)**：列表項與按鈕之間應保持至少 **8px** 的間距，避免誤觸。
- **狀態反饋**：所有按鈕點擊後必須有明顯的視覺反饋（如顏色變深或 Loading 狀態），以應對現場網路延遲。

### 表單設計
- **輸入框大小**：Input 欄位的高度建議設定為 **44px - 54px**。
- **選擇器 (Selectors)**：在進行狀態選擇（如保養等級）時，應優先使用 **Radio Buttons** 或 **Segmented Controls**，而非下拉選單 (Dropdown)，以減少點擊次數。

## 2. 視覺與佈局 (Visual & Layout)

### 佈局策略
- **響應式設計**：使用 Tailwind CSS 的 , ,  斷點，確保在 iPad 的橫向 (Landscape) 與縱向 (Portrait) 模式下皆能完美呈現。
- **單欄佈局**：在行動端/平板端，盡量採用單欄佈局，避免複雜的多欄表格。

### 色彩與易讀性
- **高對比度**：考慮現場照明環境，文字與背景應保持高對比度（符合 WCAG AA 標準）。
- **狀態色彩**：
    - 成功/完成 $\rightarrow$ 綠色 ()
    - 警告/待辦 $\rightarrow$ 黃色 ()
    - 錯誤/緊急 $\rightarrow$ 紅色 ()

## 3. 離線與性能 (Offline & Performance)

- **本地快取 (Local Cache)**：針對現場網路不穩定的情況，重要的表單資料應先暫存於  或  中，待網路恢復後再行同步。
- **圖片上傳**：拍照上傳時，應在前端先進行圖片壓縮 (Compression)，減少上傳時間與頻寬消耗。
